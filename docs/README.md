# ODD Toolkit Documentation

This directory contains user-facing documentation for the ODD Toolkit extensibility features.

## Documentation Files

### QUICKSTART.md
A 5-minute quick start guide with:
- Build and run instructions
- Basic CLI usage examples
- Creating your first custom generator in 5 steps
- Quick reference for available generators

**Start here if you're new to the extensibility features**

### cli-guide.md
Comprehensive CLI usage documentation covering:
- All available generators and their descriptions
- Running generators from command line
- Configuration file formats (YAML and JSON)
- CLI argument reference
- Environment variable reference
- 10+ real-world use case examples
- Troubleshooting and debugging

**Read this if you want to use generators from the command line**

### guide/ontology-metadata.md
Comprehensive reference for ontology design and semantic metadata:
- Supported OWL 2 constructs
- Hydra vocabulary for API metadata
- Dublin Core and SKOS metadata
- SHACL constraint definitions
- Ontology structure best practices
- Complete e-commerce ontology example
- Validation and troubleshooting

**Read this when designing ontologies or working with semantic metadata**

### extension-guide.md
Complete guide for extending the toolkit with custom generators and adapters:
- Architecture overview
- Step-by-step generator creation guide
- Step-by-step adapter creation guide
- Component registration approaches
- Complete PlantUML generator example
- Unit testing patterns
- Best practices and design patterns

**Read this if you want to create custom generators or adapters**


## Getting Started

1. **For first-time users:** Start with [QUICKSTART.md](QUICKSTART.md)
2. **For CLI usage:** See [cli-guide.md](cli-guide.md)
3. **For extending the toolkit:** See [extension-guide.md](extension-guide.md)

## Documentation Structure

All documentation uses:
- Relative paths (e.g., `src/main/java/...`)
- Generic instructions suitable for any developer
- No personal or temporary paths
- Focus on user needs and use cases
